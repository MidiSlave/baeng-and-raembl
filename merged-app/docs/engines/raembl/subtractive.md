# Ræmbl SUB (Subtractive) Synthesis Engine

The **SUB (Subtractive)** engine is Ræmbl's classic analogue-style synthesis engine, featuring anti-aliased oscillators, dual TPT filters, and dedicated ADSR envelopes. It delivers warm, rich tones reminiscent of legendary hardware synthesisers like the Roland TB-303, Moog minimoog, and Sequential Prophet-5.

## Overview

The SUB engine implements **classic subtractive synthesis**: rich harmonic waveforms are generated by oscillators, blended together in the mixer, then sculpted by filters and shaped by envelopes. The result is a versatile synthesis engine capable of everything from punchy basslines to lush pads.

### Key Features

- **5 waveforms**: Sawtooth, triangle, square, sub-oscillator, white noise
- **PolyBLEP anti-aliasing**: Eliminates aliasing artefacts at all pitches
- **TPT filters**: Topology-Preserving Transform lowpass and highpass filters with resonance
- **Independent ADSR envelopes**: Amplitude and filter envelopes with independent parameters
- **Dual LFO modulation**: Main LFO (pitch/PWM) and Mod LFO (filter/pitch/PWM)
- **TB-303 features**: Slide (glide), trill, accent for authentic acid bassline performance
- **8-voice polyphony**: Pre-allocated AudioWorklet voices with intelligent voice stealing

### Best Use Cases

- **TB-303 style acid basslines** - Classic squelchy resonant bass
- **Analogue leads** - Screaming mono synth leads with filter sweeps
- **Warm pads** - Lush polyphonic textures with slow filter movement
- **Traditional subtractive synthesis** - Anything you'd do with a classic analogue synth

## Signal Path

The SUB engine follows this audio signal chain:

```
┌──────────────┐
│ OSCILLATORS  │  5 waveforms: Saw, Tri, Square, Sub, Noise
│ (PolyBLEP)   │  + Pulse Width Modulation (PWM)
│              │  + Octave transposition (independent main/sub)
│              │  + Drift (pitch instability)
└──────┬───────┘
       │
       ▼
┌──────────────┐
│    MIXER     │  Blend oscillator levels (0-100% each)
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   HIGHPASS   │  TPT highpass filter (20Hz - 10kHz)
│   FILTER     │  + Fixed resonance (0.7)
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   LOWPASS    │  TPT lowpass filter (20Hz - 20kHz)
│   FILTER     │  + Resonance (0.1 - 25)
│              │  + Envelope modulation
│              │  + Key tracking
│              │  + LFO modulation
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   ENVELOPE   │  Amplitude ADSR envelope
│   (AMP)      │  + Velocity sensitivity
│              │  + Accent boost (1.5x)
└──────┬───────┘
       │
       ▼
    OUTPUT
```

**Processing Notes:**
- Filters are processed in series (highpass → lowpass)
- All modulation is **k-rate** (30 FPS on main thread, not audio-rate)
- AudioWorklet voices are pre-allocated (8 voices, zero creation overhead)

## Oscillators

The SUB engine provides **5 simultaneous waveforms** with independent level control and PolyBLEP anti-aliasing.

### Waveform Types

| Waveform | Symbol | Character | Harmonics | Best For |
|----------|--------|-----------|-----------|----------|
| **Sawtooth** | ◢ | Bright, buzzy, rich | All harmonics (1/n amplitude) | Basses, leads, strings |
| **Square** | ⊓ | Hollow, woody, nasal | Odd harmonics only | Clarinets, hollow leads |
| **Triangle** | △ | Mellow, soft, flute-like | Odd harmonics (1/n² amplitude) | Flutes, soft pads |
| **Sub** | ■ | Deep, fundamental-only | Pure sine wave | Sub-bass reinforcement |
| **Noise** | ≋ | White noise (all frequencies) | Non-harmonic | Breath, percussion, texture |

### PolyBLEP Anti-Aliasing

All waveforms (except sub-oscillator and noise) use **PolyBLEP** (Polynomial Band-Limited Step) anti-aliasing to eliminate aliasing artefacts:

- **Sawtooth/Square**: PolyBLEP corrects amplitude discontinuities (step functions)
- **Triangle**: PolyBLAMP (integrated PolyBLEP) corrects slope discontinuities

**Result**: Clean waveforms at all pitches, even at high frequencies near Nyquist limit.

### Oscillator Parameters

| Parameter | Range | Default | PPMod | Description |
|-----------|-------|---------|-------|-------------|
| **OCT** | 0-8 | 4 | ✓ | Main oscillator octave transposition (0 = C-2, 4 = C3, 8 = C7) |
| **SUB OCT** | 0-8 | 2 | ✓ | Sub-oscillator octave transposition (**independent** from main OCT) |
| **DRIFT** | 0-100 | 10 | ✓ | Pitch instability amount (±40 cents max at 100%) |
| **GLIDE** | 0-100 | 0 | ✓ | Portamento time (0 = instant, 100 = 5 seconds) |
| **WIDTH** | 5-95% | 50% | ✓ | Pulse width for square wave (5% = narrow, 50% = square, 95% = wide) |
| **PWM** | 0-100 | 0 | ✓ | Pulse width modulation depth (from Mod LFO) |
| **MOD** | 0-100 | 0 | ✓ | Pitch modulation depth (from Mod LFO, ±1 semitone max) |

**Important Notes:**

- **OCT and SUB OCT are independent**: You can transpose the main oscillators and sub-oscillator separately (e.g., main at C4, sub at C2 for classic sub-bass)
- **Drift is per-note random**: Each note gets a random pitch offset within the drift range, emulating analogue oscillator instability
- **GLIDE vs SLIDE**: GLIDE is a global portamento parameter; SLIDE is a per-step sequencer flag triggering 80ms TB-303 style glide

### Pulse Width Modulation (PWM)

The square wave oscillator supports **pulse width modulation** (PWM), creating chorus-like movement and stereo width.

**Modulation Sources:**
- **Mod LFO**: `PWM` parameter controls depth (0-100)
- **Filter Envelope**: Routed via `envToPWM` AudioParam (0-100)

**Range**: ±40% modulation (clamped to 5%-95% to prevent silence)

**Example**: `WIDTH=50%`, `PWM=50%` → width sweeps between 30% and 70% in sync with Mod LFO

## Mixer

The **MIXER** module blends the 5 oscillator waveforms. Each waveform has an independent level control (0-100%).

### Mixer Parameters

| Parameter | Range | Default | PPMod | Description |
|-----------|-------|---------|-------|-------------|
| **◢ (SAW)** | 0-100 | 75 | ✓ | Sawtooth oscillator level |
| **⊓ (SQR)** | 0-100 | 0 | ✓ | Square/pulse oscillator level |
| **△ (TRI)** | 0-100 | 0 | ✓ | Triangle oscillator level |
| **■ (SUB)** | 0-100 | 50 | ✓ | Sub-oscillator level (sine wave) |
| **≋ (NOISE)** | 0-100 | 0 | ✓ | White noise level |

**Mixing Strategy:**
- Levels are **additive** (not crossfading) - you can have all 5 waveforms active simultaneously
- Total level is **uncompressed** - high combined levels may clip (use Output VOL to compensate)
- **Recommended starting point**: 75% saw, 50% sub, 0% others (classic TB-303 style)

### Classic Oscillator Combinations

| Sound | Saw | Square | Tri | Sub | Noise | Description |
|-------|-----|--------|-----|-----|-------|-------------|
| **Classic 303 Bass** | 100 | 0 | 0 | 50 | 0 | Pure saw + sub reinforcement |
| **Hollow Lead** | 0 | 100 | 0 | 0 | 0 | Square wave only (clarinet-like) |
| **Warm Pad** | 50 | 0 | 50 | 30 | 0 | Saw + triangle blend |
| **Breathy Flute** | 0 | 0 | 75 | 0 | 25 | Triangle + noise |
| **Aggressive Lead** | 75 | 75 | 0 | 0 | 0 | Saw + square (rich harmonics) |

## Filter Section

The SUB engine uses **TPT (Topology-Preserving Transform)** filters, a modern digital filter design that maintains stability at all cutoff frequencies and resonance values.

### Filter Architecture

**Dual serial filters:**
1. **Highpass filter** - Removes low-frequency rumble (fixed 0.7 resonance)
2. **Lowpass filter** - Shapes tone and removes high-frequency content (variable resonance)

**Filters are processed in series**: Signal passes through highpass first, then lowpass.

### Filter Parameters

| Parameter | Range | Default | PPMod | Description |
|-----------|-------|---------|-------|-------------|
| **HP** | 0-100 | 0 | ✓ | Highpass filter cutoff (0 = 20Hz, 100 = 10kHz) |
| **LP** | 0-100 | 75 | ✓ | Lowpass filter cutoff (0 = 20Hz, 100 = 20kHz) |
| **RES** | 0-100 | 20 | ✓ | Lowpass filter resonance/Q (0 = gentle, 100 = self-oscillating) |
| **KEY** | 0-100 | 0 | ✓ | Keyboard tracking amount (0 = fixed cutoff, 100 = 1:1 pitch tracking) |
| **ENV** | 0-100 | 50 | ✓ | Filter envelope modulation depth (bipolar: 50 = neutral) |
| **MOD** | 0-100 | 0 | ✓ | Filter modulation depth (from Mod LFO, ±1 octave) |

### Filter Modulation Routing

The lowpass filter cutoff can be modulated by **three sources**:

#### 1. Filter Envelope (ENV)
- **Bipolar control**:
  - `ENV < 50`: Negative modulation (-2 octaves max at 0%)
  - `ENV = 50`: No modulation (neutral)
  - `ENV > 50`: Positive modulation (+4 octaves max at 100%)
- **Use case**: Classic filter sweeps (e.g., TB-303 "squelch")

#### 2. Keyboard Tracking (KEY)
- **Scales cutoff with note pitch**: Higher notes → higher cutoff
- **Tracking ratio**: At 100%, filter tracks pitch 1:1 (cutoff doubles per octave)
- **Use case**: Maintain consistent brightness across the keyboard

#### 3. Mod LFO (MOD)
- **Cyclic modulation**: ±1 octave range (bipolar LFO)
- **Rate controlled by**: Mod LFO RATE parameter (0.1-20 Hz)
- **Waveform**: Sine/Triangle/Square (selected via Mod LFO WAVE)
- **Use case**: Auto-wah, cyclic filter movement

**Combined Modulation Formula:**
```javascript
finalCutoff = baseCutoff
              × keyTrackingMultiplier
              × 2^(envMod + lfoMod)
```

### Resonance Behaviour

- **0-20%**: Gentle resonance boost at cutoff frequency
- **20-60%**: Pronounced resonance peak (classic analogue sound)
- **60-90%**: Aggressive, self-oscillating behaviour (ringing/whistling)
- **90-100%**: Extreme self-oscillation (can produce pure sine wave at cutoff)

**Warning**: High resonance (>80%) + high envelope modulation can produce loud transients. Monitor levels carefully.

## Envelopes

The SUB engine has **two independent ADSR envelopes**: amplitude and filter.

### Envelope Parameters

| Parameter | Range | Default | PPMod | Description |
|-----------|-------|---------|-------|-------------|
| **A (Attack)** | 0-100 | 0 | ✓ | Attack time (0 = 0.2ms, 100 = 8 seconds, exponential) |
| **D (Decay)** | 0-100 | 25 | ✓ | Decay time (0 = 0.2ms, 100 = 8 seconds, exponential) |
| **S (Sustain)** | 0-100 | 50 | ✓ | Sustain level (0 = silent, 100 = full level) |
| **R (Release)** | 0-100 | 75 | ✓ | Release time (0 = 0.2ms, 100 = 8 seconds, exponential) |

**Important**: Attack and release parameters are **independent** - changing attack does not affect release timing (this was a bug in earlier versions, now fixed).

### Amplitude Envelope

Controls the **output volume** over time.

**Stages:**
1. **Attack**: Note on → ramp to full volume (attack time)
2. **Decay**: Full volume → ramp to sustain level (decay time)
3. **Sustain**: Hold at sustain level (while note is held)
4. **Release**: Note off → ramp to silence (release time)

**Accent Effect:**
- **1.5× velocity boost**: Accented notes are louder
- **2× faster attack**: Creates punchy transient (TB-303 style)
- **0.5× decay time**: Snappier, tighter envelope

### Filter Envelope

Controls the **filter cutoff modulation** over time (routed via `ENV` parameter).

**Behaviour:**
- Same ADSR stages as amplitude envelope
- **Independent timing**: Filter envelope can have different attack/decay/release than amplitude
- **Routing**: Envelope output is scaled by `ENV` parameter before modulating filter cutoff

**Classic Uses:**
- **Pluck**: `A=0, D=40, S=0, R=20` + `ENV=70%` → filter closes after attack
- **Sweep**: `A=30, D=60, S=20, R=40` + `ENV=80%` → slow filter opening
- **Static**: `ENV=50%` → no filter modulation (cutoff stays constant)

### Envelope Snapshot on Release

When a note is released, the envelope **snapshots its current value and release time**. This ensures:
- **Parameter changes don't affect releasing voices** (e.g., changing release time won't cut off voices already ringing out)
- **Consistent release behaviour** regardless of when parameters are tweaked

## Modulation (LFO)

The SUB engine has **two LFO systems**: Main LFO (PATH module) and Mod LFO (MOD module).

### Main LFO (PATH Module)

**Not used by the subtractive engine directly** - this LFO modulates the pitch path sequencer and is part of the generative sequencing system.

### Mod LFO (MOD Module)

**Dedicated modulation source** for oscillator and filter parameters.

| Parameter | Range | Default | PPMod | Description |
|-----------|-------|---------|-------|-------------|
| **RATE** | 0-100 | 50 | ✓ | LFO frequency (0 = 0.1Hz, 100 = 20Hz) |
| **WAVE** | Sine/Tri/Square/Saw | Sine | ✓ | Waveform selection (click to cycle) |

**Modulation Destinations:**

1. **Pitch Modulation** (`OSC → MOD` parameter)
   - Range: ±1 semitone
   - Use case: Vibrato, pitch wobble

2. **Pulse Width Modulation** (`OSC → PWM` parameter)
   - Range: ±40% pulse width
   - Use case: Chorus-like movement, stereo width

3. **Filter Modulation** (`FILTER → MOD` parameter)
   - Range: ±1 octave
   - Use case: Auto-wah, cyclic filter sweeps

**Waveform Characteristics:**

- **Sine**: Smooth, natural modulation (vibrato, auto-wah)
- **Triangle**: Linear ramps (cyclic sweeps)
- **Square**: Stepped modulation (trill-like effects, tremolo)
- **Sawtooth**: Asymmetric ramps (rising/falling sweeps)

## Mono vs Poly Mode

The SUB engine supports **monophonic** and **polyphonic** playback modes, each with distinct voice allocation behaviour.

### Mode Selection

Switch modes using the **M/POLY** toggle in the oscillator module:
- **M (Mono)**: Monophonic mode
- **P2**: 2-voice polyphonic mode
- **P4**: 4-voice polyphonic mode (default)
- **POLY8**: 8-voice polyphonic mode (maximum)

### Monophonic Mode (M)

**One voice active at a time** - new notes steal the active voice.

**Key Features:**

1. **Legato**: Notes played while holding another note **do not retrigger the envelope**
   - Envelope continues from current stage
   - Only pitch changes (smoothly via glide)
   - Classic synth behaviour for expressive playing

2. **Glide (Portamento)**: Smooth pitch transitions between notes
   - **GLIDE parameter** (0-100) controls glide time
   - Exponential pitch ramp from previous note to new note
   - Works in legato and non-legato playing

3. **Slide**: TB-303 style glide **without envelope retrigger**
   - Triggered by **SLIDE** flag in sequencer
   - **Fixed 80ms glide time** (authentic TB-303 timing)
   - Envelope continues from current state (no attack phase)

**Typical Use Cases:**
- **Basslines**: Classic monophonic TB-303 style acid
- **Leads**: Expressive solos with pitch bends and glides
- **Sequenced patterns**: Euclidean patterns with slide/trill

### Polyphonic Mode (P2/P4/POLY8)

**Multiple voices active simultaneously** - new notes allocate new voices (up to limit).

**Voice Allocation Strategy:**

1. **Find inactive, non-releasing voice** (cleanest allocation)
2. **If none available, steal voice based on stealability score**:
   - **Age**: Older voices are more stealable
   - **Velocity**: Quieter voices are more stealable
   - **Accent/Trill**: Accented or trilled notes are **protected** from stealing

**Polyphonic Slide Behaviour:**
- **40ms "slide-into" effect** on note trigger (gentler than mono mode's 80ms)
- Each voice glides independently
- Envelope **always retriggers** (no legato in poly mode)

**PPMod in Poly Mode:**
- **8 independent modulation instances** (one per voice)
- **Phase offset**: Voices start at different LFO phases to prevent unison (0°, 45°, 90°, 135°, 180°, 225°, 270°, 315°)
- **Per-voice filter modulation**: Each voice's filter sweeps independently

**Typical Use Cases:**
- **Chords**: Harmonic progressions
- **Pads**: Lush polyphonic textures
- **Arpeggios**: Multiple simultaneous notes with independent modulation

## Sound Design Tips

### Creating Classic TB-303 Acid Basslines

**Recipe:**
1. **Engine**: SUB
2. **Waveform**: 100% **SAW** (◢) or 75% Saw + 50% Square (⊓)
3. **Filter**:
   - **LP**: 30-50%
   - **RES**: 60-80% (high resonance for squelch)
4. **Envelope**:
   - **A**: 0 (instant attack)
   - **D**: 40 (quick decay)
   - **S**: 0 (no sustain - pluck-like)
   - **R**: 20 (short release)
5. **Filter Envelope**: **ENV**: 70% (positive filter modulation)
6. **Sequencing**:
   - **STEPS**: 16, **FILLS**: 8-12
   - Add **SLIDE** on upward pitch movements
   - Add **ACCENT** on downbeats and syncopated notes
   - Use **Minor Pentatonic** or **Minor** scale
7. **Effects**: Delay 25%, Reverb 10-20%

**Advanced Technique**: Modulate **LP** (filter cutoff) with **PPMod LFO** (slow rate ~0.5 Hz, 30% depth) for evolving filter sweeps.

---

### Creating Warm Analogue Pads

**Recipe:**
1. **Engine**: SUB
2. **Polyphony**: **POLY4** or **POLY8**
3. **Waveform**: 50% **SAW** + 50% **TRI** + 30% **SUB**
4. **Filter**:
   - **LP**: 60-70% (gentle lowpass)
   - **RES**: 10-20% (subtle resonance)
   - **KEY**: 40% (slight keyboard tracking for consistent brightness)
5. **Envelope**:
   - **A**: 60 (slow attack)
   - **D**: 40 (medium decay)
   - **S**: 80 (high sustain)
   - **R**: 70 (long release)
6. **Filter Envelope**: **ENV**: 55% (slight positive modulation for movement)
7. **Modulation**:
   - **PPMod LFO** on **LP**: Slow rate (0.3 Hz), 15% depth, sine wave
   - **PPMod LFO** on **DRIFT**: Very slow rate (0.1 Hz), 20% depth (adds shimmer)
8. **Effects**: Reverb 40%, Delay 15% (SYNC mode, 1/4 note)

**Tip**: Use **Major**, **Lydian**, or **Maj Pent** scale for bright, uplifting pads.

---

### Creating Screaming Mono Leads

**Recipe:**
1. **Engine**: SUB
2. **Polyphony**: **M (Mono)**
3. **Waveform**: 100% **SAW** or 75% Saw + 75% Square
4. **Filter**:
   - **LP**: 70-80% (bright starting point)
   - **RES**: 50-70% (pronounced resonance)
   - **KEY**: 80% (filter follows pitch closely)
5. **Envelope**:
   - **A**: 0-10 (fast attack)
   - **D**: 30 (quick decay)
   - **S**: 60 (medium sustain)
   - **R**: 30 (short release)
6. **Filter Envelope**: **ENV**: 75% (strong positive filter sweep)
7. **Modulation**:
   - **GLIDE**: 30 (smooth pitch transitions)
   - **Mod LFO → MOD**: 50 (±1 semitone vibrato)
   - **Mod LFO RATE**: 60 (moderate vibrato speed ~6 Hz)
8. **Effects**: Delay 30%, Reverb 20%

**Performance Tip**: Use **SLIDE** in sequencer for expressive pitch glides between notes.

---

### Creating Percussive Plucks

**Recipe:**
1. **Engine**: SUB
2. **Polyphony**: **POLY4** (for chords)
3. **Waveform**: 75% **SAW** + 50% **TRI**
4. **Filter**:
   - **LP**: 50% (mid-range starting point)
   - **RES**: 30% (moderate resonance for "ping")
5. **Envelope**:
   - **A**: 0 (instant attack)
   - **D**: 25 (quick decay)
   - **S**: 0 (no sustain - pluck dies out)
   - **R**: 30 (short release)
6. **Filter Envelope**: **ENV**: 80% (strong positive modulation - filter closes quickly)
7. **Filter Env Timing**:
   - **A**: 0 (instant)
   - **D**: 20 (very quick decay)
   - **S**: 10 (low sustain)
   - **R**: 40 (medium release)
8. **Effects**: Reverb 25%, Delay 10% (adds space)

**Variation**: Add **NOISE** at 15% for plectrum/pick noise.

---

### Creating Aggressive Sync Leads

**Recipe:**
1. **Engine**: SUB
2. **Polyphony**: **M (Mono)**
3. **Waveform**: 100% **SAW** + 75% **SQUARE**
4. **Filter**:
   - **LP**: 85% (very bright)
   - **RES**: 75% (aggressive resonance)
   - **KEY**: 90% (filter tracks pitch closely)
5. **Envelope**:
   - **A**: 5 (snappy attack)
   - **D**: 35 (medium decay)
   - **S**: 70 (high sustain)
   - **R**: 25 (quick release)
6. **Filter Envelope**: **ENV**: 70% (positive modulation)
7. **Modulation**:
   - **PWM**: 60 (pulse width modulation for movement)
   - **Mod LFO RATE**: 70 (~8 Hz)
   - **Mod LFO WAVE**: Square (stepped modulation)
8. **PPMod**: **RND mode** on **LP** (16-bit LFSR, 80% probability, 20% depth) for unpredictable filter movement
9. **Effects**: Delay 20%, Saturation (via Clouds FB parameter if in Clouds mode)

**Tip**: Use **ACCENT** on key notes for extra punch.
